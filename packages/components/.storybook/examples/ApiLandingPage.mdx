## 4. Pricing — usage-based

Pay for exactly what you use, with a free-tier buffer before metering starts. The cost formula is published in full so you can estimate ahead of shipping:

<BlockMath math={String.raw`\text{cost}(n) = \max\left(n_\text{billable} \cdot \$0.001,\; \$5.00\right)`} />

where <InlineMath math={String.raw`n_\text{billable} = \max(0,\; n_\text{requests} - 10{,}000)`} /> — the first 10 000 requests each month are on the house.

### Example bills

<CodeBlockStorybook
  lang="ts"
  code={`// 8,000 requests this month → free tier covers it
cost(8_000) === $5.00   // floor

// 25,000 requests → 15,000 billable
cost(25_000) === max(15_000 * 0.001, 5) === $15.00

// 1M requests → 990,000 billable
cost(1_000_000) === max(990_000 * 0.001, 5) === $990.00`}
/>

## 5. Rate limits — token bucket

Every API key gets a token bucket. Each request consumes one token; tokens refill at a steady rate. The maximum concurrent burst is the bucket depth:

<BlockMath math={String.raw`\text{tokens}_{t+1} = \min\left(\text{capacity},\; \text{tokens}_t + r \cdot \Delta t\right) - 1`} />

Defaults:

- **Capacity** — <InlineMath math={String.raw`\text{capacity} = 120`} /> tokens (burst)
- **Refill rate** — <InlineMath math={String.raw`r = 2`} /> tokens/sec
- **Sustained** — 120 req/min steady-state

Exceed the bucket and you get a `429 Too Many Requests` with a `Retry-After` header. Contact us for higher limits.

## 6. Webhooks — streaming status updates

Long-running generations stream partial tokens via SSE, and status updates to your webhook endpoint:

<CodeBlockStorybook
  lang="json"
  code={`{
  "event": "poem.completed",
  "created": 1726838492,
  "data": {
    "id": "poem_abc123",
    "status": "completed",
    "tokensUsed": 58,
    "body": "A single rose blooms —\\nsunlight tracing each soft petal,\\nthen gone. Still, it was."
  }
}`}
/>

Verify every webhook with the `X-Poet-Signature` header before acting. Worked example:

<CodeBlockStorybook
  lang="ts"
  code={`import crypto from "node:crypto";

export function verifyWebhook(body: string, sig: string, secret: string): boolean {
  const expected = crypto
    .createHmac("sha256", secret)
    .update(body)
    .digest("hex");
  return crypto.timingSafeEqual(
    Buffer.from(sig, "hex"),
    Buffer.from(expected, "hex"),
  );
}`}
/>
