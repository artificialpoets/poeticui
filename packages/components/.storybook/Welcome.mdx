# Poetic UI

**Neutral React primitives that consume design tokens — zero brand assumptions, ready for open source.**

Poetic UI is split across four layers so a consumer app can pick only the pieces it needs:

Everything in this Storybook is the **middle layer** — neutral, brand-agnostic. The theme switcher in the toolbar flips between the default light/dark neutral themes.

- **Content** — Code blocks, math, package-manager + language tab strips. See the [overview](?path=/docs/content-overview--docs) or the [API Landing Page](?path=/docs/content-examples-api-landing-page--docs) showcase for a realistic end-to-end page.
- **Core / Data Display / Forms / Feedback / Layout / Navigation / Misc / Tables** — the components, grouped by role

Every component has a `Default` story, one story per meaningful variant, and a `VariantMatrix` for at-a-glance combinatorial review.

## Authoring new components

Read [`CONTRIBUTING.md`](https://github.com/artificialpoets/poeticui/blob/main/packages/components/CONTRIBUTING.md) for the full contract. TL;DR:

1. **Semantic tokens only** — no raw color utilities. `bg-card`, not `bg-white`.
2. **`cx()` not `clsx`** — `cx` layers tailwind-merge so consumer `className` wins on conflicts.
3. **CVA for variants** — Pattern A (single `cva()`) or Pattern B (separate color lookup).
4. **Slot composition over monolith props** — families like `<Fieldset>` → `<FieldGroup>` → `<Field>` → `<Label>`.
5. **HeadlessUI wrappers are canonical** — import our `Dropdown`, not `@headlessui/react` `Menu`.
6. **Charts stay outside the library** — chart components (`BarChart`, `AreaChart`, etc.) live in the dashboard because they need data adapters. Chart Recipes doc shows the patterns.
